17. Explain the difference between absolute and relative URLs when creating links.
  • Absolute URL
  • An absolute URL provides the complete address to a resource on the web.
  • It includes the protocol (http:// or https://), domain name, and the complete file path.
  • Example:
  • About Us
  • Characteristics
  • Points to a resource on a different website or a fully-qualified link on the same site.
  • Useful when linking to external sites.
  • Always works regardless of the current location of the page.
  • Relative URL
  • A relative URL provides a path relative to the current page's location.
  • It does not include the domain name or protocol.
  • It can be a path to a file in the same directory, a subdirectory, or a parent directory.
  • Examples:
  • About Us
  • > About Us
  • About Us

  • Characteristics

  • and easier to manage within a site.
  • If you move the whole site, relative links still work.
  • Depends on the current location of the HTML file.
